# Code of Conduct

## Our standard

This project is meant to be useful, direct, and low-drama.

People participating here are expected to:
- be respectful and constructive
- assume good intent, then get specific about problems
- keep feedback focused on the work
- be honest about validation, uncertainty, and tradeoffs
- avoid hostility, harassment, dogpiling, or personal attacks

## Unacceptable behavior

This includes, but is not limited to:
- harassment, threats, or discrimination
- insulting or demeaning language aimed at a person or group
- bad-faith review behavior, trolling, or repeated escalation
- sharing private information without permission
- knowingly dishonest claims about testing, authorship, or repo state

## Scope

This applies to GitHub issues, pull requests, discussions, and other project spaces where people are collaborating around this repository.

## How to handle problems

If someone is making the project worse for others:
1. pause the escalation
2. point to the specific behavior
3. ask for a reset or course correction
4. if needed, remove or block the interaction

Maintainers may edit, hide, lock, or remove content that violates this code of conduct.

## Enforcement philosophy

Use judgment.

The goal is not maximum policy text. The goal is a repo where people can contribute useful work without dealing with avoidable nonsense.
